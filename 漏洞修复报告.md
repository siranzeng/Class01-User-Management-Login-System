## 修复问题清单

### 1. /profile 越权访问（高危）

**问题：** /profile 路由完全没有 session 登录校验和权限校验。攻击者无需登录即可通过修改 URL 参数 `user_id` 遍历所有用户的个人资料（含邮箱、手机、余额）。

**漏洞代码（修复前）：** `app.py` 第 386-409 行

```python
@app.route("/profile")
def profile():
    user_id = request.args.get("user_id", "")
    user = None
    if user_id and user_id.isdigit():
        conn = sqlite3.connect("data/users.db")
        c = conn.cursor()
        c.execute("SELECT id, username, email, phone FROM users WHERE id = ?", (user_id,))
        row = c.fetchone()
        conn.close()
        if row:
            uid, uname, email, phone = row
            user_data = USERS.get(uname)
            if user_data:
                user = {
                    "id": uid, "username": uname,
                    "email": email, "phone": phone,
                    "balance": user_data.get("balance", 0),
                    "role": user_data.get("role", "user")
                }
    return render_template("profile.html", user=user)
```

**修复：** 增加 session 登录校验，并增加权限判断：普通用户只能查看自己的资料，管理员可查看全部。

```python
if "username" not in session:
    return redirect("/login")

target_uid = int(user_id)
if target_uid != current_data["id"] and current_data.get("role") != "admin":
    error = "无权查看其他用户的资料"
```

**POC 验证结果：**

| 测试场景 | 修复前 | 修复后 |
|----------|--------|--------|
| 未登录访问 /profile | 正常返回任意用户资料 | 302 重定向到 /login |
| alice 查看 admin 资料 | 正常返回 admin 信息 | 提示"无权查看其他用户的资料" |
| admin 查看 alice 资料 | 正常返回 alice 信息 | 正常返回（管理员权限） |
| 遍历 user_id=1~100 | 批量枚举全部用户信息 | 非管理员被拒绝 |

### 2. /recharge 越权操作（高危）

**问题：** /recharge 路由完全没有 session 校验和权限校验。攻击者无需登录即可通过构造 POST 请求给任意用户充值（或通过负数金额盗刷余额）。

**漏洞代码（修复前）：** `app.py` 第 412-429 行

```python
@app.route("/recharge", methods=["POST"])
def recharge():
    user_id = request.form.get("user_id", "")
    amount = request.form.get("amount", "0")
    if user_id and user_id.isdigit():
        conn = sqlite3.connect("data/users.db")
        c = conn.cursor()
        c.execute("SELECT username FROM users WHERE id = ?", (user_id,))
        row = c.fetchone()
        conn.close()
        if row:
            username = row[0]
            if username in USERS:
                USERS[username]["balance"] += float(amount)
    return redirect(f"/profile?user_id={user_id}")
```

**修复：** 增加 session 登录校验，并限制非管理员只能给自己充值。

```python
if "username" not in session:
    return redirect("/login")

target_uid = int(user_id)
if target_uid != current_data["id"] and current_data.get("role") != "admin":
    error = "无权操作其他用户的充值"
```

**POC 验证结果：**

| 测试场景 | 修复前 | 修复后 |
|----------|--------|--------|
| 未登录充值 | 成功充值 | 302 重定向到 /login |
| alice 给 admin 充值 | 成功增加 admin 余额 | 拒绝并提示错误 |
| admin 给 alice 充值 | 成功增加 alice 余额 | 成功（管理员权限） |
| 批量遍历充值 user_id=1~100 | 全部成功 | 非管理员被拒绝 |

### 3. 负数金额盗刷余额（高危）

**问题：** amount 参数未做任何正负校验，攻击者可传入负数值减少他人余额（盗刷），或传入极大值使余额溢出。

**漏洞代码（修复前）：** `app.py` 第 416、427 行

```python
amount = request.form.get("amount", "0")      # 无正负校验
USERS[username]["balance"] += float(amount)    # 直接累加
```

**修复：** 增加 amount 正数校验，拒绝非正数值。

```python
try:
    amount_val = float(amount)
except ValueError:
    amount_val = 0

if amount_val <= 0:
    error = "金额必须为正数"
    return redirect(f"/profile?user_id={user_id}&error={error}")
```

**POC 验证结果：**

| 测试场景 | 修复前 | 修复后 |
|----------|--------|--------|
| amount=-500 负数充值 | 余额成功扣减 500 | 拒绝："金额必须为正数" |
| amount=0 零充值 | 余额不变但无提示 | 拒绝："金额必须为正数" |
| amount=abc 非法输入 | ValueError 500 错误 | 拒绝："金额必须为正数" |
| amount=200 正常充值 | 余额 +200 | 余额 +200 |

### 4. 导航栏硬编码越权链接（中危）

**问题：** base.html 导航栏"个人中心"链接硬编码为 `/profile?user_id=1`，导致所有用户点击后默认查看管理员资料。

**漏洞代码（修复前）：** `templates/base.html` 第 16 行

```html
<a href="/profile?user_id=1" class="navbar-link">个人中心</a>
```

**修复：** 改为 `/profile`（不带参数），路由自动使用当前登录用户 ID。

```html
<a href="/profile" class="navbar-link">个人中心</a>
```

```python
# app.py 中
user_id = request.args.get("user_id", str(current_data["id"]))
```
