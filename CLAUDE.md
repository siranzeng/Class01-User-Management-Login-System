# 项目开发规范

## 每次提交必须更新 README.md

1. **新增功能或修改代码后，必须同步更新 README.md**
2. 在 `## 版本历史` 章节追加新版本记录
3. 更新 `## 漏洞修复记录` 表格
4. 修改的文件清单需要列在对应版本说明中

## README 更新模板

新增版本时按以下格式追加：

```markdown
### v2.x — 版本标题

**新增功能：**
- 功能1
- 功能2

**修复漏洞：**

| 漏洞 | 描述 | 修复方式 |
|------|------|----------|

**修改文件：** 文件名1, 文件名2
```
